divide(/) is an expensive operation

2016-03-07

I am working with this problem this afternoon.

At first I wrote an algorithm whose idea is same with the standard solution, but what returned by OJ was TLE…
then I tried to check my code with another correct solution line by line and nothing was different except I used
a large number of divide operations, but I didn’t believe that will make such a big difference.
Without any other choices, I rewrite my code by merging several divide operations. e.g.
rewrite
$$ a/n+b/n+c/n $$
into
$$ (a+b+c)/n $$
.
And finally it got Accepted…

What we should learn from this problem is that contant time complexity operations really matters when time is limited or tight.
