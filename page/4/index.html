Problems Link

Problem A: greedily take more problems as you can. suppose we are now at i’th problem, check whether we can take 4 continous problems start at position i, if we cann’t, check 3, 2, 1 in the same way.
Problem B: using priority queue to keep trace of the time that every washer can finish one load clothes after finishing all former given clothes and initially queue was populated with pairs of (w[i], i) for each washer. In this way we can manage to finish all load of clothes as early as possible. we use the same way to dryers.
Problem C: attention that initial money coder has is a real number not a integer.
since choosing uniformly at random from the range [A, B], so the probability of every number to be chosen is equal to $\frac{1}{B−A}$. so now what we need is the left money for all initial savings we may choose. It’s not difficult to find that the equation consists of a series segments and our answer is total area of trangles formed by those segments with x-axis. finally we can compute area from 0 to A and 0 to B respectively.
Problem D: using bitmask dynamic programming.
since factorial of 16 is too large so we cannot try every permutation. but you will notice there only five round at most because we choose one from every pair competitors each round that is the placing a competitor can have is in (1, 2, 3, 5, 9).
in addition, if a competitor can beat every other competitors than he will always end up with first placing. otherwise we can make he lose at the first round with placing $\lfloor n \rfloor + 1$.
using dp[s][i] represents the i’th competitor can be a winner if he fights with competitors whose coresponding bit is set in integer s (the number of bits set in s can only be 1 or 2 or 4 or 8 or 16). Initially, dp[1<<i][i] is always true because there is only one competitor in this state and he is the winner without doubt. If we have dp[s][i] = dp[t][j] = true and s & t = 0 and win[i][j] = true than dp[s|t][i] = true otherwise dp[s|t][j] = true.
The best placing for each competitor can be calculated by the largest subset of fixed size(1,2,4,8,16) he can win.

Probelms Link

Problem D: using dynamic programming.
When printing the chosen list of words, operations are saved when consecutive words have common prefixes. As such, it can be shown that an optimal solution can always be produced by printing words in lexicograghical order.
After sorting all N words, we must decide which K of them to use. In order to calculate the number of operations required to print a word, we only care about the previous word that was printed. As such, this problem can be solved with dynamic programming, with the state DP[i][c] = minimum number of operations required to print c words, the last of which was word i. For each initial word i, DP[i][1] = L[i] + 1 (where L[i] is the length of the ith word). The final answer will then be the smallest value DP[i][K] + L[i], for some final word i.
Another thing need to be done is pre-calculate common prefix between the ith word and the jth word, we precompute LCP[i] = the length of common prefix of words i and i + 1, and to achieve what we need can using a simply RMQ technique.

MO's Algorithm

2016-04-07

MO’s Algorithm is used to deal with problems without update operations. The idea of MO’s Algorithm is to pre-process all queries so that result of one query can be used in next query.
Blow are steps:

  1. Sort all queries in a way that queries with L values from 0 to $\sqrt{n}−1$ are put together, then all queries from $\sqrt{n}$ to $2 \times \sqrt{n}−1$, and so on.
  2. Process all queries one by one in a way that every query uses sum computed in previous query.
    • Let ‘ans’ be answer of previous query.
    • Remove extra elements of previous query and update answer.
    • Add new elements of current query and update answer.
      here attention that you should make sure it only costs $O(1)$ that is constant time (but sometimes $O(log ⁡)$$ is also tolerable) to update the answer when add or remove one element Here are some basic problems can be solved using MO’s Algorithm.

Pick’s Theorem
Pick’s Theorem states that if we have a polygon with lattice points as vertices, then:
$$A=I+B/2−1$$

where A is the area of the polygon, I is the number of lattice points inside of the polygon, and B is the number of lattice points on the boundary of the polygon.
BTW, we can find B by noting that the number of points that lie on a line with lattice endpoints (W,X) and (Y,Z) is
$$1+gcd(|Y−W|,|Z−X|)$$.
